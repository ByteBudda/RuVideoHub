# Техническая документация: Профессиональный медиа-плеер для Android

Этот документ представляет собой максимально детальное описание архитектуры, функциональности и принципов работы медиа-плеера для Android. Приложение спроектировано как высокопроизводительный, модульный инструмент для воспроизведения медиаконтента (потокового или локального) с уникальной функцией синхронизации субтитров в реальном времени.

---

## 1. Обзор архитектуры

Приложение построено на основе принципов **Clean Architecture** с использованием паттерна **MVVM (Model-View-ViewModel)** и декларативного UI-фреймворка **Jetpack Compose**. Такая структура обеспечивает строгое разделение ответственности, высокую масштабируемость и легкость тестирования.

### Основные слои:
1.  **Слой данных (Data Layer)**: Определение моделей данных.
2.  **Слой логики (Domain/Manager Layer)**: Обработка медиа-ресурсов, логика синхронизации субтитров, работа с сетью, управление загрузками.
3.  **Слой представления (UI Layer)**: Декларативное описание интерфейса и управление состоянием плеера.

---

## 2. Детальный разбор компонентов

### 2.1. Плеерный движок: `ExoPlayerHandler`
`ExoPlayerHandler` — это центральный компонент, инкапсулирующий работу с медиа-движком `Media3` (ExoPlayer).

*   **Управление жизненным циклом**: Контролирует процесс от инициализации плеера до корректного освобождения системных ресурсов (`release()`).
*   **Инициализация**: Конфигурирует параметры буферизации (`DefaultLoadControl`), подготавливает `MediaItem` и устанавливает исходные треки субтитров.
*   **Движок синхронизации субтитров**:
    *   **Асинхронность**: Использует `CoroutineScope` и `Dispatchers.Main` для безопасного управления задачами.
    *   **Обработка**: При изменении задержки отменяет текущую задачу сдвига, запускает новую в фоновом потоке, корректирует тайминги, создает временный файл в кэше и перезагружает `MediaItem` без прерывания воспроизведения (используя `seekTo`).

### 2.2. Движок обработки данных: `SubtitleUtils`
Модуль (Singleton), реализующий алгоритмы трансформации данных субтитров. Это сердце работы с текстовыми данными.

*   **Парсинг таймингов**: Преобразует строковое представление времени (различные форматы) в абсолютное значение миллисекунд (`Long`).
*   **Математика сдвига**: Итеративно проходит по сегментам, применяет математический сдвиг (`start + delay`, `end + delay`) для каждого временного интервала, обеспечивая целостность (время не может быть отрицательным).
*   **Форматирование**: Выполняет обратное преобразование миллисекунд в строковое представление для записи в итоговый файл, адаптируясь под формат исходного файла.

### 2.3. Модели данных
Определяют структуру треков субтитров, истории просмотра и закладок. Это обеспечивает консистентность данных между API, локальной БД (`Room`) и UI.

### 2.4. Пользовательский интерфейс и ViewModel
*   **`VideoViewModel`**: Удерживает состояние экрана воспроизведения, связывает данные из репозитория с UI.
*   **Компоненты Compose**: Отображают видео, подписываются на состояние из ViewModel и реагируют на действия пользователя.
*   **Абстракция источников (Плагины)**: Инкапсулирует логику работы с различными внешними источниками контента. Это позволяет легко добавлять поддержку новых источников, не меняя основной логики плеера.

## 3. Технологический стек

| Технология | Назначение |
| :--- | :--- |
| **Kotlin** | Основной язык разработки. |
| **Jetpack Compose** | Декларативный UI, реактивное управление состоянием. |
| **Media3 (ExoPlayer)** | Ядро воспроизведения медиа-потоков. |
| **Kotlin Coroutines / Flow** | Асинхронное выполнение задач, реактивные потоки данных. |
| **Room Database** | Локальная БД для сохранения контента. |
| **Retrofit / OkHttp** | Сетевое взаимодействие. |

## 4. Сборка и интеграция

### Системные требования:
*   Android Studio (Flamingo или выше).
*   JDK 17+.

### Сборка:
1.  Убедитесь, что все зависимости в `build.gradle.kts` (Media3, Coroutines, OkHttp) настроены.
2.  Разрешения в `AndroidManifest.xml` (необходим доступ к сети):
    ```xml
    <uses-permission android:name="android.permission.INTERNET" />
    ```
3.  Для сборки debug-версии используйте:
    ```bash
    ./gradlew assembleDebug
    ```
